## ВСТУП

### Мета та завдання роботи

**Мета:** Розроблення веб-орієнтованої системи управління музейними виставками, новинами та інформацією про відвідувачів з метою автоматизації роботи адміністраторів музею та підвищення якості надання інформації посвідувачам.

**Завдання:**
1. Проаналізувати вимоги до системи управління музеєм
2. Спроєктувати архітектуру системи на основі паттерну MVC
3. Розроблити модуль управління виставками з функціями фільтрації та пошуку
4. Реалізувати модуль управління новинами з підтримкою пагінації
5. Реалізувати систему аутентифікації для адміністраторів
6. Створити RESTful API для доступу до даних системи
7. Провести тестування всіх функціональних модулів
8. Підготувати документацію користувача та розробника

### Предмет дослідження

Предметом дослідження є методи та технології розроблення веб-орієнтованих систем управління контентом на базі PHP та MySQL з використанням MVC архітектури та RESTful принципів.

### Об'єкт дослідження

Об'єктом дослідження є система управління музейними виставками та новинами, яка включає:
- Backend компоненти (API, контролери, моделі)
- Frontend компоненти (представлення, форми)
- Модуль аутентифікації та авторизації
- Базу даних з таблицями виставок, новин, користувачів
- Систему файлозбереження для зображень

### Актуальність роботи

Актуальність даної роботи обґрунтовується наступним:

1. **Цифрова трансформація музейної сфери** - Музеї активно впроваджують цифрові технології для покращення взаємодії з відвідувачами та оптимізації внутрішніх процесів.

2. **Необхідність автоматизації** - Традиційні методи управління виставками та контентом потребують значних часових витрат та можуть призвести до помилок.

3. **Збільшення відвідуваності** - Веб-платформи дозволяють музеям досягти більшої аудиторії та підвищити залученість відвідувачів.

4. **Актуальність технологій** - PHP та MySQL залишаються найпопулярнішими технологіями для розроблення веб-додатків, особливо в контексті малих та середніх проєктів.

5. **Освітня цінність** - Проект демонструє практичне застосування принципів веб-розроблення, включаючи MVC, RESTful API, ORM та безпеку.

### Практичне значення роботи

Практичне значення роботи полягає у:

1. **Готова система до впровадження** - Розроблена система може бути адаптована та впроваджена в реальних музеях для управління їх контентом.

2. **Шаблон для подальшого розвитку** - Архітектура та код можуть бути використані як основа для розроблення аналогічних систем в інших установах.

3. **Економія часу адміністраторів** - Система дозволяє значно скоротити час, необхідний для управління виставками та новинами.

4. **Покращення користувацького досвіду** - Відвідувачі отримують можливість зручно переглядати виставки, фільтрувати їх за типом та датою, читати новини.

5. **Розширюваність** - Модульна архітектура дозволяє легко добавляти нові функції, такі як коментарі, рейтинги, резервування квитків.

6. **Навчальний матеріал** - Проект може бути використаний як навчальний матеріал для студентів, які вивчають веб-розроблення.

## РОЗДІЛ 1. ОПИС ТЕХНОЛОГІЧНОГО СТЕКУ

### 1.1 Огляд технологій

Система управління музейними виставками розроблена з використанням сучасного технологічного стеку, який поєднує проверені рішення з гнучкістю та масштабованістю.

**Backend:** Проект використовує PHP 7.4+ як мову програмування, що забезпечує швидкість розроблення та широку підтримку хостинг-провайдерів. Для організації коду реалізовано власне MVC-фреймворк, що складається з таких компонентів:

- **Core клас** управляє марутизацією запитів та ініціалізацією додатку
- **Controller** базовий клас для обробки HTTP-запитів
- **Model** базовий клас для роботи з даними та БД
- **DB** клас з методами для SELECT, INSERT, UPDATE, DELETE запитів
- **Template** рушій для рендерингу HTML-шаблонів
- **Request** клас для обробки даних з форм та URL
- **Session** клас для управління сеансами користувача

**База даних:** MySQL 5.7+ використовується для зберігання структурованих даних. Структура включає таблиці для виставок, новин, типів виставок, зображень, користувачів та відвідувачів. Підтримка транзакцій та індексування забезпечують високу продуктивність.

**Frontend:** HTML5 забезпечує семантичну розмітку, CSS3 використовується для адаптивного дизайну, JavaScript (ES6+) додає інтерактивність. Система повністю响應式та працює на мобільних пристроях.

**API:** RESTful API реалізовано за принципами HTTP, з використанням методів GET для отримання даних. JSON обраний як основний формат обміну даними. API підтримує фільтрацію, пошук та пагінацію.

**Безпека:** Для запобігання SQL-інєкції використовуються підготовлені твердження (prepared statements) через PDO. Паролі користувачів хешуються за допомогою встроєних PHP функцій. Сеанси контролюються та автоматично завершуються після неактивності.

**Розгортання:** Apache з модулем mod_rewrite для переписування URL-адрес. Файли проекту організовані за принципом розділення логіки (classes/, controllers/, models/, views/) для легкого обслуговування та масштабування.

### 1.2 Архітектура системи

Система побудована на тришаровій архітектурі:

1. **Presentation Layer** (шар представлення) - HTML шаблони та CSS стилі для відображення інформації користувачам
2. **Business Logic Layer** (шар бізнес-логіки) - Controllers, які обробляють запити та підготовляють дані
3. **Data Access Layer** (шар доступу до даних) - Models та DB клас для роботи з БД
4. **Data Layer** (шар даних) - MySQL база даних

Така архітектура дозволяє легко модифікувати кожен шар без впливу на інші.

### 1.3 Основні компоненти

**Модуль управління виставками** дозволяє адміністраторам створювати, редагувати та видаляти виставки, завантажувати зображення, встановлювати дати проведення. Для відвідувачів передбачена можливість переглядати виставки з фільтрацією за типом та датою, пошуком за назвою.

**Модуль управління новинами** забезпечує створення та публікацію новин з можливістю додавання текстового контенту, зображень та дати публікації. Новини відображаються в зворотному хронологічному порядку з пагінацією по 6 записів на сторінці.

**Модуль аутентифікації** реалізує вхід адміністраторів до системи з перевіркою креденціалів та управлінням сеансами. Ролева система (admin/user) забезпечує контроль доступу до різних частин системи.

**API** надає REST ендпоінти для доступу до даних: GET /exhibitions/show, GET /exhibitions/view/{id}, GET /news/show, GET /news/view/{id}, GET /site/contacts. Всі відповіді повертаються у форматі JSON з інформацією про статус, дані та помилки.
